> ここが歪むと MoneyForward のインポートが弾かれる＝EDDの合否条件。確定日 2026-06-28。

## ファイル仕様

- **エンコーディング：Shift-JIS（CP932）**。UTF-8/utf-8-sig は不可（文字化けで弾かれる）。
- ヘッダ行あり（27列・順序厳守）。
- **1件の正本JSON → 1行**（税率が単一のとき。複数税率は方式Bで除外）。
- 取引日：`YYYY/M/D`（ゼロ埋めなし・スラッシュ）。例 `2026/4/12`。
- 金額：整数（カンマなし）。借方金額 = 貸方金額 = `document.total`（税込）。

## 借方税区分の導出（税率 → 表記。空白込みで厳密一致）

- `tax.rate_10` のみ（rate_8 が 0/不在）→ `課税仕入 10%`
- `tax.rate_8` のみ → `課税仕入 8%`
- **rate_10 と rate_8 が両方 > 0（複数税率混在）→ 方式B：CSVに出さず 02_エラー へ**（warning `複数税率混在`）

> 「課税仕入 10%」は半角スペースを含む。MF現物の正書法に一致させること（**最終照合は要確認**）。

## 検証（出力前・EDD）

- `document.total > 0`、`document.date` が `YYYY-MM-DD` で解釈可能。
- 税合算＝合計：`rate_10.base+rate_10.amount + rate_8.base+rate_8.amount == total`（`validation.tax_sum_matches_total`）。
- `status == "failed"` の JSON は CSV に出さない（エラー扱い）。
- いずれか不成立 → CSV化せず 02_エラー へ（推測補完しない）。
