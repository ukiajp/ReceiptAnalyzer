# 勘定科目蒸留表の作り方（構成B・ADR-014）

> MFの仕訳帳エクスポートCSVから「店名 → 借方勘定科目」の参照表を作る手順。
> 確定スクリプトが店名から決定論的に勘定科目を付与するための参照データ。改善時はこの手順を更新する。
> 参照表の置き場：`99_勘定科目参照/勘定科目参照表_v1.md`（受信トレイと同じドライブ）。

---

## 入力データ

- ソース：MoneyForward「仕訳帳」エクスポートCSV（例 `仕訳帳_YYYYMMDD_HHMM.csv`）。Shift-JIS(CP932)。
- **注意：仕訳帳エクスポートは19列形式**で、インポート用CSV（27列）とは別物。列順：
  `1取引No 2取引日 3借方勘定科目 4借方補助科目 5借方部門 6借方取引先 7借方税区分 8借方インボイス 9借方金額 10貸方勘定科目 … 17摘要 18タグ 19メモ`

## 抽出条件（ノイズ除去）

- 借方税区分（7列）に「課税仕入」を含む（費用の購入行に絞る）。
- 摘要（17列）が空でない（店名がある行だけ）。
- → 期首残高（対象外・摘要空）や振替が落ちる。

## 店名キーの正規化

- 決済手段サフィックス除去：`/iD` `/NFC` `/QUICPay` `/nanaco` `/楽天Edy` `/QP`
- 連番除去：`@数字`、`モバイルオーダー`、末尾空白
- 既知の改善余地：支店名の揺れ（`阪神西宮店`/`梅田本店`）、英字大小、表記違いは現状別キー。

## 集計と確信ヒント

- 正規化キーごとに借方勘定科目の出現数を数え、最多を主勘定科目とする。
- 確信：高＝件数5以上かつ主科目8割以上 / 中＝6割以上 / 低＝割れ or 件数3未満（→ 決め打ちしつつ人確認）。

## 出力フォーマット（参照表）

Markdownテーブル。列＝`店名キー / 主勘定科目 / 確信 / 件数 / 内訳`。
スクリプトはこの表を読み、`partner_normalized`（または partner_name）を正規化して**部分一致は「キーが店名に含まれる」単方向＋完全一致優先**でマッチ（誤マッチ防止）。

## 再生成（bash・要点）

```bash
f="仕訳帳_YYYYMMDD_HHMM.csv"
iconv -f CP932 -t UTF-8 "$f" | awk -F'","' '
NR>1 { acct=$3; tax=$7; tekiyo=$17;
  if (tax ~ /課税仕入/ && tekiyo != "") {
    key=tekiyo; gsub(/\/(iD|NFC|QUICPay|nanaco|楽天Edy|QP)/,"",key);
    gsub(/@[0-9]+/,"",key); gsub(/モバイルオーダー/,"",key); gsub(/[ 　]+$/,"",key);
    cnt[key SUBSEP acct]++; tot[key]++; } }
END { for (k in cnt){ split(k,a,SUBSEP); s=a[1]; ac=a[2]; c=cnt[k];
    if (c>maxc[s]){maxc[s]=c; main[s]=ac}; brk[s]=brk[s](brk[s]?" / ":"")ac":"c }
  for (s in tot){ r=maxc[s]/tot[s];
    conf=(tot[s]>=5 && r>=0.8)?"高":((r>=0.6)?"中":"低"); if(tot[s]<3)conf="低";
    printf "%d\t%s\t%s\t%s\t%s\n", tot[s],s,main[s],conf,brk[s] } }' \
  | sort -t$'\t' -k1,1nr -k2,2 > distill.tsv
# → distill.tsv を Markdown表（店名キー|主勘定科目|確信|件数|内訳）に整形して 99_勘定科目参照/勘定科目参照表_v1.md へ
```
